Problem

App 路径已经使用 session.* 接口（Interface），但实现（Implementation）仍承担旧 IM route 的复杂度。

Solution

围绕 sessionId 与 SessionRecorder 加深 App Conversation Module；App 发送不再创建 app ChatRef 状态。

Benefits

  • locality: 一条发送路径
  • leverage: sessionId 接口
  • 测试穿过 SessionRecorder
  • route 泄漏收缩

Top recommendation

先做候选 1，再删除 chat.send。

先让 App session 路径不再依赖 IM seam。这样后续每一步删除都有 locality：飞书可以消失，chat.send 可以消失，route_bindings 也可以在不扰动 Conversation Registry session flow 的情况下移除。
