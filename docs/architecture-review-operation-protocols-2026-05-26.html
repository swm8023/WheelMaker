Workspace Operation Runtime

Problem. The current App interface to long operations is shallow: every view knows timer cleanup, pending flags, polling cadence, and stale response guards.

Solution. Move operation state, timeout, polling, serial policy, and cancellation into one Workspace Operation Runtime module, with domain adapters for search, uploads, npm, skills, and update.

  • locality: timers concentrate
  • leverage: views shrink
  • interface absorbs races

Problem. Callers must learn a different interface for each long operation even when the implementation concept is the same.

Solution. Define one operation contract vocabulary and let existing method families adapt into it without flattening their domain payloads.

  • leverage: one UI contract
  • locality: status labels
  • interface gets smaller

Top recommendation

Start with the Registry Method Registry: it has an immediate correctness bug, high locality, and gives the operation runtime a stable seam to build on.
