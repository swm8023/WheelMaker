1. 需要增加一个配置文件config.json用于配置，也在home下，首先计入project的概念，有个name，project里要配im和client，im可以配比如console,feishu等等，feishu的化还要配appID和secretID用于连接，client可以配置agent,比如codex,claude等，还有path也很重要。除了project外，还有一些通用配置，比如feishu_config，设置一些参数，可以暂定。
2. 可以支持同时管理多个client，可以加个hub？启动的时候hub会启动所有client，连接im等。state的格式要同时改下，以project为单位，
3. im增加console类型，就是从控制台获取，当有project的im为console时，启动后进入监听这个console project的输入模式，用于测试用。
4. client到agent可以lazy加载，收到第一条消息的时候才创建，避免占用。超时比如半小时没有任何信息的收发，也可以关掉agent并存盘，下次发消息恢复即可。
5. 有/就一定是command么，万一代码一开始就是/，是不是不匹配做message输入即可
6. console的配置有个debug参数，有的话要打印所有acp的往来json
