# WheelMaker App — AI Agent Guide

Flutter mobile app (iOS / Android) that connects to the WheelMaker Go daemon via WebSocket,
providing a chat interface to remotely control local AI CLIs.

## Architecture at a Glance

```
ConnectScreen → WsService (WebSocket) → Go daemon
ChatScreen    → subscribes WsService.messages stream
                            ↓
                    options card + FilledButton.tonal
                            ↓ user taps
                    selectOption(decisionId, optionId)
```

## Key Files (quick map)

| Path | Role |
|------|------|
| `lib/services/ws_service.dart` | WebSocket connect, auth handshake, send/receive |
| `lib/screens/chat_screen.dart` | Main chat UI, options card rendering |
| `lib/screens/connect_screen.dart` | Server address + token input |
| `lib/models/chat_message.dart` | `ChatMessage` / `MessageType` / `OptionItem` |

## WsState Machine (quick ref)

```
disconnected → connecting → authenticating → ready → disconnected
                                             ⇘ error
```

## App-specific Constraints

- Flutter SDK ≥ 3.2.0
- Platform dirs (`android/`, `ios/`) are generated by `flutter create`, never hand-written
- `WsService` owns all WebSocket state; screens are stateless consumers of its streams

## Dev Quick Start

```bash
# in app/
flutter pub get
flutter run                    # connect a device first
flutter build apk --debug
```
